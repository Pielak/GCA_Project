Configuração de SMTP

Protocolo de envio de e-mails para credenciais e convites

1
PRIMEIRA
Pendente
Por que é a primeira prioridade? SMTP é a base da comunicação do sistema. Sem ele, o Admin não consegue enviar credenciais aos GPs e os GPs não conseguem convidar membros da equipe. Configure SMTP antes de qualquer outra coisa.
Importante para sua empresa
Solicite à sua área de TI/Infraestrutura as credenciais SMTP do servidor corporativo. Se usar Gmail, gere uma Senha de Aplicativo (não use sua senha pessoal).
Gmail: smtp.gmail.com | Corporativo: Solicitar à TI
TLS: 587 | SSL: 465
→ Gerar Senha de Aplicativo (Gmail)
Copie exatamente como fornecido (16 caracteres com espaços)
Se deixar em branco, enviará para o e-mail de envio configurado acima
Rotação de Credenciais: Altere a senha de aplicativo a cada 90 dias por segurança.

Integração de Repositório

Personal Access Token para sincronização automática de código no repositório

2
SEGUNDA
Pendente
Por que é a segunda prioridade? Com SMTP funcionando, agora o GP consegue acessar o sistema. Ele precisa sincronizar o código gerado no GitHub automaticamente. O PAT (Personal Access Token) do Admin permite isso.
→ Gerar PAT (Scopes: repo, user) | Válido por 90 dias | Não compartilhe com ninguém
Será preenchido automaticamente após validação
Rotação de Credenciais: Revogue e gere um novo PAT a cada 90 dias. Nunca reutilize PATs antigos.

Documentação do Admin Setup

Bem-vindo ao Painel de Administração do GCA

Este painel foi desenvolvido para facilitar a configuração inicial do sistema GCA com segurança e clareza.

Como Usar

  1. Configure cada seção na ordem indicada (1-5)
  2. Preencha os campos obrigatórios com as credenciais dos serviços
  3. Clique em "Testar" para validar a configuração
  4. Ao passar no teste, clique em "Validar" para prosseguir
  5. Ao concluir todas as 5 seções, o setup estará completo

Segurança

  • Nunca compartilhe suas credenciais (API Keys, Tokens, Senhas)
  • Altere suas credenciais a cada 90 dias
  • Use senhas fortes e únicas para cada serviço
  • Revogue credenciais antigas imediatamente

Dica: Consulte o arquivo ADMIN_SETUP_GUIDE.md para documentação completa com mais detalhes sobre cada etapa.
