# FASE 7: GitHub Integration

## Overview

FASE 7 implements complete GitHub integration for managing repositories, creating pull requests, and automating code deployment workflows. Generated code from FASE 4 can now be automatically committed and submitted as pull requests to GitHub repositories.

## Architecture

### Components

1. **GitHubIntegration** (`app/services/github_service.py`)
   - GitHub API v3 client wrapper
   - Token-based authentication
   - Repository management
   - Branch and commit operations
   - Pull request management

2. **GitHubWorkflow** (`app/services/github_service.py`)
   - High-level workflow orchestration
   - Complete deployment pipeline
   - Automated PR creation

3. **GitHubRouter** (`app/routers/github.py`)
   - REST API endpoints for GitHub operations
   - User information retrieval
   - Repository management
   - Branch and commit operations
   - PR management

## REST API Endpoints

### 1. Validate GitHub Token

```http
POST /api/v1/github/validate
Content-Type: application/json

{
  "access_token": "ghp_..."
}
```

**Response**:
```json
{
  "valid": true,
  "message": "GitHub token is valid"
}
```

### 2. Get Authenticated User

```http
POST /api/v1/github/user
Content-Type: application/json

{
  "access_token": "ghp_..."
}
```

**Response**:
```json
{
  "login": "pielak",
  "name": "Luiz Carlos Pielak",
  "email": "user@example.com",
  "id": 12345,
  "avatar_url": "https://avatars.githubusercontent.com/u/12345"
}
```

### 3. List User Repositories

```http
POST /api/v1/github/repositories

{
```

### 4. Create Branch

```http
POST /api/v1/github/branches/create
Content-Type: application/json

{
  "owner": "pielak",
  "repo": "gca",
  "branch_name": "gca/generated-feature-20260405",
  "source_branch": "main"
}
```

**Response**:
```json
{
  "name": "gca/generated-feature-20260405",
  "sha": "abc123def456...",
  "ref": "refs/heads/gca/generated-feature-20260405"
}
```

### 5. Create Commit

```http
POST /api/v1/github/commits/create
Content-Type: application/json

{
  "owner": "pielak",
  "repo": "gca",
  "branch": "gca/generated-feature-20260405",
  "file_path": "generated/code.py",
  "content": "def hello():\n    print('Hello')",
  "commit_message": "GCA: Generated code for ecommerce"
}
```

**Response**:
```json
{
  "sha": "def789ghi012...",
  "message": "GCA: Generated code for ecommerce",
  "url": "https://github.com/Pielak/GCA/commit/def789ghi012",
  "file": "code.py"
}
```

### 6. Create Pull Request


**Response**:

```
     ↓

```

## Configuration

### Required
- **GitHub Personal Access Token** (classic)
  - Scopes needed: `repo`, `user`
  - Generate at: https://github.com/settings/tokens

### Environment Variables
```bash
# For authentication, pass token in request headers or body
# No env vars needed - tokens passed per-request for security
```

## Security Considerations

### Token Management
✅ **Never commit tokens** - Pass via request headers or body
✅ **Use Personal Access Tokens** - Not OAuth
✅ **Minimal scopes** - Only `repo` and `user` needed
✅ **Rotate periodically** - Good practice

### API Rate Limits
```
Authenticated: 5,000 requests/hour
Per-repo: Create branches/commits/PRs have specific limits
Strategy: Implement exponential backoff
```

### Authentication
```
All GitHub endpoints require token parameter (query param or header)
Pass as: ?token=ghp_... or Authorization: token ghp_...
```

## Use Cases

### Case 1: Auto-Deploy Generated Code
```python
# Generate code in FASE 4
code = await code_generation_service.generate_project_code(...)

# Validate in FASE 6
validation = await validation_service.validate_code(code, ...)

# Deploy to GitHub in FASE 7
if validation["is_deployable"]:
    deployment = await github_workflow.deploy_generated_code(
        owner="pielak",
        repo="gca",
        project_name="ecommerce",
        generated_code=code
    )
```

### Case 2: Review Generated Code
```python
# After PR is created, add comment with review
await github.add_comment_to_pr(
    owner="pielak",
    repo="gca",
    pr_number=42,
    comment="Code generation complete ✅\n\nMetrics:\n- Quality: 95%\n- Security: 100%"
)
```

### Case 3: Track Code Generation
```python
# Get all PRs created by GCA
prs = await github.get_pull_requests(
    owner="pielak",
    repo="gca",
    state="all"
)

# Filter for GCA-created PRs
gca_prs = [pr for pr in prs if "GCA:" in pr["title"]]
```
